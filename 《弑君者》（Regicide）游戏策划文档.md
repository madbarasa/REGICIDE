 # 《弑君者》（Regicide）游戏策划文档
## Game Design Document (GDD)

 以下是完整的《弑君者》游戏策划文档：

---

## 1. 游戏概述 (Game Overview)

### 1.1 基础信息
- 游戏名称: 弑君者 (Regicide)
- 游戏类型: 卡牌合作/单人桌游
- 平台: 实体卡牌游戏（扑克牌可玩）/ 数字版移植
- 目标玩家: 1-4人（建议2-4人最佳体验）
- 游戏时长: 20-40分钟
- 难度等级: 高（策略+运气）

### 1.2 核心循环
玩家轮流对抗12个Boss（J、Q、K各4个花色），通过打出扑克牌造成伤害，同时管理手牌防御Boss攻击，最终击败所有Boss获得胜利。

---

## 2. 核心玩法 (Core Gameplay)

### 2.1 游戏结构
```
[设置阶段] -> [回合循环] -> [Boss战] -> [胜利/失败判定]
```

### 2.2 牌堆结构（关键配置）

#### 2.2.1 Boss牌库（12张）
| 层级 | 卡牌 | 数量 | 放置顺序 |
|------|------|------|----------|
| 顶层 | J (Jack) | 4张 | 最先放置，随机洗牌后放顶部 |
| 中层 | Q (Queen) | 4张 | 随机洗牌后放在J之上 |
| 底层 | K (King) | 4张 | 放在最底部 |

技术实现: 
- 使用栈结构管理Boss牌库
- 初始状态：K(底) -> Q(中) -> J(顶)
- 每花色内随机排序，但层级固定

#### 2.2.2 玩家牌库（酒馆）
- 包含：40张标准扑克，A, 2-10（不含大小王），每种花色各1张，共40张牌
- 感化机制: Boss被感化（精确击杀）后加入玩家牌库，成为强力伙伴牌

### 2.3 Boss属性配置表

| Boss类型 | 血量(HP) | 攻击力(ATK) | 免疫花色 | 特殊说明 |
|----------|----------|-------------|----------|----------|
| J (Jack) | 20 | 10 | 对应花色 | 基础Boss |
| Q (Queen)| 30 | 15 | 对应花色 | 中等难度 |
| K (King) | 40 | 20 | 对应花色 | 最终Boss |

注: 所有Boss免疫与其花色相同的玩家花色技能（详见3.2.1）

### 2.4 回合流程（状态机设计）

```
TURN_START
    ↓
[阶段1: 打出卡牌]
    - 选择：从手牌中选择任意张数卡牌 或 放弃进攻。
    - 系统自动验证组合有效性（单张/宠物组合/连招）。
    - 记录回合行为计数（ActionCount）。
    ↓
[阶段2: 激活花色技能]
    - 根据打出牌的花色触发对应效果。
    - 方片技能采用序列化循环补给（Active -> Next...）。
    ↓
[阶段3: 造成伤害]
    - 计算总伤害 = 基础伤害 + 技能加成（草花翻倍）。
    - 判定：是否击败Boss？
        - HP ≤ 0: 进入清理或感化逻辑，进入新 Boss 阶段。
        - HP > 0: 进入阶段 4。
    ↓
[阶段4: 承受反击与防御结算]
    - Boss 发动反击。
    - **防御豁免 (v2.9.0)**: 
        - 若本回合 ActionCount = 1，玩家必须弃牌防御。
        - 若本回合 ActionCount > 1 且选择放弃进攻，则触发“防御豁免”，直接跳过反击进入下一位玩家的回合。
    - 正常防御结算：弃牌点数 ≥ Boss 当前 ATK。
    ↓
TURN_CONTINUE / END
    - 防御成功后，ActionCount 累加，当前玩家继续回合。
    - 仅在显式执行“放弃出牌”且处理完可能的反击后，ActionCount 重置，切换下家。
```

## 3. 系统机制 (Systems)

### 3.1 手牌管理系统

#### 3.1.1 手牌上限表
| 玩家人数 | 手牌上限 | 是否加入Joker（目前未启用这条规则，下同） |
|----------|----------|---------------|
| 1人 | 8张 | 是（2张） |
| 2人 | 7张 | 否 |
| 3人 | 6张 | 否 |
| 4人 | 5张 | 是（可选） |

#### 3.1.2 抽牌规则
- 游戏开始时：所有玩家抽满手牌上限
- 回合中抽牌：仅通过方片技能触发
- 牌库抽空：游戏失败

### 3.2 花色技能系统（核心机制）

#### 3.2.1 技能效果表
| 花色 | 效果名称 | 效果描述 | 技术实现要点 |
|------|----------|----------|--------------|
| ♣ 草花(Clubs) | 暴击 | 伤害翻倍 | 最终伤害 = 牌面值 × 2 |
| ♠ 黑桃(Spades) | 护盾 | 削减 Boss 攻击力。打出后留在护盾区。 | 缩小至原尺寸 1/3，累加削减值。 |
| ♦ 方片(Diamonds) | 补给 | 玩家全队循环按序抽牌。 | 当前玩家优先，溢出则按特质回流。 |
| ♥ 红心(Hearts) | 复活 | 将弃牌堆顶的N张牌（N=牌面值）洗入牌库底部 | 先判定是否达到"顶"，避免越界 |

#### 3.2.2 免疫机制
- 触发条件: 当前Boss花色 = 打出牌花色
- 效果: 该花色的技能完全失效（仅保留基础伤害）
- 示例: 黑桃Boss在场时，打出黑桃牌只有基础伤害，无护盾效果

### 3.3 组合技系统

#### 3.3.1 宠物机制 (A牌)
- 触发条件: A牌与单张牌组合打出
- 效果: 
  - 数值 = A(1) + 主牌数值
  - 获得两种花色的所有技能
- 限制: A牌不能参与连招（见3.3.2）
- 示例: ♦8 + ♣A → 数值9，抽9张牌，造成18点伤害

#### 3.3.2 连招机制
- 触发条件: 打出多张同数值的牌（2-10）
- 限制条件: 合计数值 ≤ 10
- 效果:
  - 数值 = 所有打出牌数值之和
  - 获得所有打出花色的技能
- 限制: A牌不能加入连招
- 合法示例: 
  - 4张2 (2+2+2+2=8)
  - 3张3 (3+3+3=9)
  - 2张5 (5+5=10)
- 非法示例: 
  - 2张6 (合计12 > 10，非法)

### 3.4 Boss战系统

#### 3.4.1 战斗状态
```typescript
BossState {
    currentBoss: Card,       // 当前Boss卡牌
    currentHP: int,          // 当前血量
    baseATK: int,            // 基础攻击力
    currentATK: int,         // 当前攻击力（受黑桃等效果影响）
    isImmune: bool,          // 是否免疫（根据花色）
}
```

#### 3.4.2 场上卡牌管理（黑桃护盾）
- 数据结构: List<Card> fieldCards
- 效果计算: currentATK = baseATK - sum(场上所有黑桃牌面值)
- 最小值限制: currentATK ≥ 0（不能为负）
- 清除时机: Boss被击败后，所有场上护盾进入弃牌堆

#### 3.4.3 胜利判定
- 击败条件: 伤害使HP ≤ 0
- 感化判定: HP = 0（精确击杀）
  - 效果: Boss加入玩家牌库顶部（可被抽到手牌中使用）
  - 视觉表现: Boss卡牌颜色/边框变化，表示"已净化"
- 普通击败: HP < 0
  - 效果: Boss进入弃牌堆

### 3.5 角色系统（Character System）

> 本节描述数字版新增的“主角系统”，在实体桌游规则基础上增加长线养成和差异化技能。

#### 3.5.1 系统目标
- 为单人模式提供**差异化开局能力**和**长期成长目标**（胜场 → 等级）。
- 通过不同角色技能改变资源流动：如控制牌库波动（吟游诗人）、中和 Boss 能力并重开起手（天选之人）。
- 保持核心规则不变，仅在**如何使用花色与手牌**上做策略增强。

#### 3.5.2 数据结构

Character {
  id: 'CHOSEN_ONE' | 'BARD' | string
  name: string
  level: 1 | 2 | 3
  portrait: string                // 立绘资源
  wins: int                       // 胜场数（从本地存档读取）
  abilities: {
    lv1: string
    lv2: string
    lv3: string
  }
  // 技能运行时参数
  skillId: 'JOKER' | 'DISCARD_DRAW' | string
  chargesLeft: int                // 当前剩余次数
  maxCharges: int                 // 等级对应最大次数
  skillResetType: 'GAME' | 'BOSS' // 重置时机
  overflowToDeck: bool            // 是否启用“回流至牌库底”的强化效果
}

### 3.5.3 等级与胜场（Progression）
胜场统计：
依据当前选中角色 id 在本地 localStorage 下记录累计胜场。
Key 示例：rgc_char_stats → { CHOSEN_ONE: 3, BARD: 1 }。
升级规则（可配表，当前实现）：
Lv1 → Lv2：累计胜场 ≥ 1
Lv2 → Lv3：累计胜场 ≥ 3
升级效果：
提升技能可用次数（charges）。
对部分角色开启额外被动（如牌堆溢出回流酒馆）。
UI 提示：在胜利结算时弹 Toast：「角色升级！当前等级: Lv.X」。
### 3.5.4 角色一：天选之人（CHOSEN_ONE）
设计定位：高风险高回报的“命运操控者”，专注于一次性重置手牌并暂时中和 Boss 能力。
等级能力（与实现中的 CHARACTERS.CHOSEN_ONE.abilities 对应）：
Lv1：
拥有 1 次 Joker 技能使用权。
Joker 效果：
当前 Boss 技能被中和（免疫等被动暂时失效）。
将当前手牌全部丢入弃牌堆，从墓地优先抽牌，凑满 8 张新手牌。
Lv2：
在 Lv1 基础上增加被动：当 ♦ 触发抽牌溢出手牌上限时，多余的抽牌不再进弃牌堆，而是“回流”至玩家牌库底（保持资源循环）。
Lv3：
Joker 技能可用次数提升至 2 次。
技能触发规则：
仅在自己回合的进攻阶段（TURN_START）可用。
使用后 chargesLeft--，直至归零不可再用。
效果对当前 Boss 生效；若切换新 Boss，其免疫状态会按常规规则重新生成。
### 3.5.5 角色二：吟游诗人（BARD）
设计定位：掌控牌库波动的节奏型角色，主打“弃牌换牌”的中等风险玩法。
等级能力（与实现中的 CHARACTERS.BARD.abilities 对应）：
Lv1【万能之手1】：
效果：在进攻阶段，玩家可以选中任意数量手牌，将其全部丢入弃牌堆，然后从牌库抽取等量新牌。
限制：每局游戏限用 1 次。
Lv2【万能之手2】：
使用次数提升为每局 2 次。
Lv3【上帝之手2】（Boss 阶段限定）：
技能重置逻辑更改为每个 Boss 战阶段最多使用 2 次：
在 nextBoss() 时，根据 skillResetType === 'BOSS' 重置 chargesLeft。
技能效果实现要点：
弃牌：从手牌数组中移除被选中的卡牌，并写入弃牌堆。
抽牌：调用与普通抽牌相同的逻辑，根据当前手牌数量和牌库剩余数量补牌。
日志与反馈：
记录弃掉的牌与抽到的新牌（例：「弃: 红心8, 方片3 | 得: 梅花10, 黑桃5」）。
播放抽牌音效，并在战斗日志中追加技能条目。

### 3.5.6 角色三：赵云 (ZHAO_YUN)
**设计定位**：高泛用性手牌，枪出如龙，七进七出！

*   **【龙胆】LV.1**：手牌中的♠可以视作♣打出，♣可以视作♠打出
    *   *说明*：打出单张或连招时，每有一张♠，会自动获得♣的暴击增益；每有一张♣，也会获得♠的护盾点数。
*   **【龙胆】LV.2**：手牌中的♥可以视作♦打出，♦可以视作♥打出
    *   *说明*：大幅增强续航，只要有任意红色牌，就可以同时触发抽牌（补给）和墓地洗回（复活）效果。
*   **【龙胆】LV.3**：所有打出卡牌无视 BOSS 免疫
    *   *说明*：彻底终结 Boss 的花色属性克制。

### 3.5.7 角色四：炼金术士 (ALCHEMIST)
**设计定位**：资源转换与延迟收益大师，将"垃圾"变黄金。

*   **【等价交换】LV.1**：每回合限 1 次，可将 1 张手牌与墓地顶 1 张牌交换。
    *   *说明*：主动从墓地“钓鱼”回收关键资源，但也存在换回更弱牌的风险。需要选中 1 张手牌后触发。
*   **【贤者之石】LV.2**：交换次数提升至 2 次，且交换后若两张牌花色相同，额外抽 1 张牌。
    *   *说明*：通过精准的花色对齐规划，实现无损甚至增益的资源轮转。
*   **【大炼成阵】LV.3**：每局限 1 次，将当前手牌和墓地牌全部洗入酒馆，从酒馆抽取至手牌上限。
    *   *说明*：绝境翻盘神技。不选中任何手牌点击技能即可触发（若本局尚未开启）。

### 3.5.8 角色五：武僧 (MONK)
**设计定位**：突破限制的连招大师，拳既是道。

*   **【震慑拳15】LV.1**：将同点数连招的点数总和限制从 10 提升至 15。
    *   *说明*：允许进行原本不被允许的高点数连招，如三张 5（15点）或两张 7（14点）。
*   **【震慑拳20】LV.2**：将连招点数限制进一步提升至 20。
    *   *说明*：允许更为恐怖的连招，如两张 10（20点）或四张 5（20点）。
*   **【震慑拳Max】LV.3**：将连招点数限制提升至 40。
    *   *说明*：质变的跨越。由于游戏单牌上限为 10 点，40 点的限制意味着你现在可以打出任意 4 张同点数的卡牌（如四张 K 或四张 10），且总和（40点）不会被系统拦截。

### 3.5.9 角色六：赌徒 (GAMBLER)
**设计定位**：纯随机性角色，用抽牌决定命运，高风险极高回报。
- **加成**：手牌上限额外增加 2 张（所有模式下）。

*   **【孤注一掷】LV.1**：每次出牌后，从酒馆随机抽出一张牌，如果花色与刚刚打出的牌相同，则加入手牌。
    *   *说明*：博弈的核心。如果你打出一张 ♠，牌库顶也是 ♠，你将免费获得它。
*   **【无限绿波】LV.2**：新增被动：每当获得手牌（包括从【孤注一掷】或方片技能中获得），从酒馆随机抽出一张牌，如果花色与刚刚获得的牌相同，则加入手牌。可以多次触发。
    *   *说明*：极致赌狗体验。
*   **【天命在我】LV.3**：【孤注一掷】和【无限绿波】的匹配条件升级为：相同花色 或 相同点数。
    *   *说明*：大幅提升命中率。从“可能赚”到“可能爆炸”。

### 3.5.10 角色七：刺客 (ASSASSIN)
**设计定位**：精准斩杀与连击爆发，强调"一击必杀"的窗口期。

*   **【背刺】LV.1**：若本回合打出的第一手攻击中包含 ♣，点数翻倍。
    *   *说明*：结合 ♣ 自身技能，首刀最高造成 4 倍伤害。
*   **【连杀】LV.2**：使用 ♣ 牌击败 Boss 后从酒馆或墓地摸 2 张牌。
    *   *说明*：确保资源的持续轮转。
*   **【死亡标记】LV.3**：标记当前 Boss，全手牌附带 ♣ 翻倍效果（每回合 1 次）。
    *   *说明*：斩杀最终武器，全场锁定，满屏 ♣。

### 3.6 Joker系统（目前已经做到一个角色的技能里了）

#### 3.6.1 单人模式专用
- 数量: 2张Joker作为"JOKER卡"（救命卡）
- 使用时机: 任意时刻
- 效果: 
  1. 将当前BOSS能力无效化（己方打出与BOSS花色一致的卡牌能力正常生效）
  2. 弃置所有手牌并抽8张新牌（可选）
  3. 翻面1张JOKER卡（标记已使用）
- 成就系统: 根据使用JOKER卡数量评级
  - 0张: S级
  - 1张: A级
  - 2张: B级

#### 3.7 多人模式 (包含AI队友与联机)
- 概念明确: **单人模式**指仅有玩家一人的对局；**多人模式**指包含 AI 队友参与的对局，以及未来将加入的真实玩家网络联机模式。
- 跳过: 玩家允许跳过（跳过行动环节，仍要承受BOSS伤害），但不允许连续跳过，即1号玩家跳过了，2号玩家就无法跳过。
- 失败判定条件: 任意一名玩家无法抵挡BOSS攻击阵亡时，一般情况下直接判定整个团队失败（除非持有复活神器）。
### 3.7.1 同步策略
- 帧同步: 不适用（回合制）
- 状态同步: 每次操作后同步GameState
- 关键同步点:
  1. 玩家出牌操作
  2. 阶段转换
  3. Boss状态变更
  4. 游戏结束判定
### 3.7.2 反作弊
- 禁止通牌: 手牌信息仅服务端和当前玩家可见
- 操作验证: 服务端校验所有操作合法性
- 随机数: 使用服务端种子洗牌，客户端预测+校正


### 3.8 神器机制（可选机制）

#### 3.8.1 神器获得
- 条件: 击败boss后，根据boss的等级，掉落的数量不同。J=1，Q=2，K=3
- 分配机制: 击败boss的玩家必定获得一件，剩余的神器按照玩家行动顺序依次自动获得。例如，3号玩家击败了bossK，则4号，1号，2号各获得一件神器，3号玩家因为行动顺序最末，未能分配到神器。
- 类型: 
  1. 战斗增益：no.001在伤害计算阶段，额外增加1点伤害/no.002手中卡牌面值+1(最大不可超过10点，可提升护盾效率)/no.003暴击伤害由2提升至3/no.004单人模式下护盾效果1.5倍(向下取整)，多人模式下基于本身全队共享性质不作额外倍率提升。

### 3.10 单机 AI 队友机制 (v2.5.0+)

#### 3.10.1 基础设定
- 配置: 玩家可在冒险模式选择 1~3 名 AI 队友，组成 2~4 人小队
- 视野隔离: 玩家无法看见 AI 手牌（UI 仅显示牌背及数量），AI 之间也互不可见，禁止作弊查牌
- 行为表现: AI 行动会有 1~2 秒拟人延迟，操作会被完整记录在战斗日志中（HUD 飘字提示后期作为设置开关）

#### 3.10.2 策略模型
AI 包含三种策略（玩家可自由分配）：平衡型、稳健型、激进型。这里以**平衡型（Balanced）** 为基础派生：

**平衡型防御逻辑（TAKE_DAMAGE 阶段）**:
1. 优先挑选 Boss 免疫的花色作为弃牌
2. 挑选数值刚好的低费用“废牌”凑防守点数
3. 最高优先级保留组合技组件（如 A）和尚未被免疫的强效技能牌（如强力护盾）

**平衡型进攻逻辑（PLAY_CARD 阶段）**:
1. 避免单出当前 Boss 免疫的花色牌（但可以用于组合/宠物）
2. 优先打出黑桃 ♠ 以削减高额攻击力（前提是不免疫）
3. 检查是否有正好“感化 (HP=0)”的解，有则优先执行击杀
4. 若无防守压力和感化可能，则尽最大努力打出高伤爆发组合（如草花 ♣ 连招）
5. 如果手牌数量 ≤ 4 张，优先使用方片 ♦ 补手牌

**派生策略**:
- **稳健型 (Defensive)**：在平衡型基础上权重偏移，更早停止进攻转而保留防守手牌，优先打出红心 ♠ 降攻或 ♥ 续航
- **激进型 (Aggressive)**：在平衡型基础上权重偏移，无视部分防御阈值，强行追求极速斩杀。

*(注：初版 AI 不会使用角色的主动技能如小丑/洗牌，后期单独加入)*

#### 3.10.3 结算机制改造预留
目前版本为：“任意一个队友防御失败，则游戏立刻结束（失败）”。代码需保持极度解耦，以便后期版本在“房间自定义规则”里增加【全军覆没才算失败】的选项。

## 7. 附录

### 7.1 术语表
- 感化: 精确击杀Boss（HP归零），Boss变为友方卡牌
- JOKER卡:救命道具
- 连招: 同数值多牌组合（合计≤10）
- 宠物: A牌的别称，可与单牌组合
- 护盾牌: 黑桃牌在场上持续生效，减少Boss攻击力
- 城堡: BOSS牌堆
- 酒馆: 玩家牌堆
- 墓地: 弃牌堆
- 玩家: 含人类玩家、AI队友及联机队友的统称
- 防御豁免: 回合内非首个动作选择放弃进攻，免受 Boss 反击。
